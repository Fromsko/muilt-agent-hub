# 状态存储方案

> 收录时间：2026-04  
> 覆盖：2026 状态管理哲学、Zustand / Jotai / Valtio / Redux Toolkit 对比、分层架构、持久化、本项目策略

---

## 一、2026 状态管理哲学

> **核心变化**：服务端状态 vs 客户端状态彻底分离

```
Layer 1: Server State    → TanStack Query (~5M 周下载)
Layer 2: Global Client   → Zustand (~4M 周下载)
Layer 3: Atomic/Derived  → Jotai (~2M 周下载)（按需）
Layer 4: Local Component → useState / useReducer
Layer 5: Form State      → React Hook Form
Layer 6: URL State       → 路由参数 / searchParams
```

大多数 2026 年新项目使用 **TanStack Query + Zustand + React Hook Form** 即可覆盖全部需求。

## 二、主流方案对比

| 维度 | Zustand | Jotai | Valtio | Redux Toolkit |
|------|---------|-------|--------|--------------|
| **心智模型** | 单一 Store（类 Redux） | 原子状态（类 Recoil） | 可变代理（类 Vue） | Slice + Reducer |
| **体积** | ~1KB | ~2KB | ~3KB | ~12KB |
| **样板代码** | 极少 | 极少 | 极少 | 中等 |
| **TypeScript** | ✅ 原生 | ✅ 原生 | ✅ 原生 | ✅ 原生 |
| **Provider** | ❌ 不需要 | ⚠️ 可选 | ❌ 不需要 | ✅ 需要 |
| **DevTools** | ✅ Redux DevTools | ✅ 自带 | ✅ proxy-compare | ✅ Redux DevTools |
| **持久化** | ✅ middleware | ✅ atomWithStorage | ✅ proxyWithPersist | ✅ redux-persist |
| **SSR** | ✅ | ✅ | ⚠️ 需注意 | ✅ |
| **re-render 优化** | selector 手动 | 原子级自动 | proxy 自动 | selector 手动 |
| **适合** | 全局 UI 状态、中大型 | 细粒度派生状态 | Vue 开发者、快速原型 | 大型企业、已有 Redux |

## 三、Zustand — 本项目主力

### 核心理念

- **一个 Store 就是一个 Hook** — `const useStore = create(...)`
- **无 Provider** — 直接在任意组件调用
- **Selector 防止无关 re-render** — `useStore(s => s.count)`
- **Middleware 可组合** — persist / devtools / immer / subscribeWithSelector

### 基础用法

```tsx
import { create } from "zustand";

interface CounterState {
  count: number;
  inc: () => void;
  dec: () => void;
  reset: () => void;
}

const useCounter = create<CounterState>((set) => ({
  count: 0,
  inc: () => set((s) => ({ count: s.count + 1 })),
  dec: () => set((s) => ({ count: s.count - 1 })),
  reset: () => set({ count: 0 }),
}));

// 使用 — selector 精确订阅
function Display() {
  const count = useCounter((s) => s.count);  // 只在 count 变化时 re-render
  return <span>{count}</span>;
}

function Controls() {
  const inc = useCounter((s) => s.inc);  // 函数引用稳定，不会 re-render
  return <button onClick={inc}>+1</button>;
}
```

### Middleware 组合

```tsx
import { create } from "zustand";
import { persist, devtools, subscribeWithSelector } from "zustand/middleware";
import { immer } from "zustand/middleware/immer";

interface AppState {
  theme: "light" | "dark" | "system";
  sidebarOpen: boolean;
  recentProjects: string[];
  setTheme: (theme: AppState["theme"]) => void;
  toggleSidebar: () => void;
  addRecentProject: (id: string) => void;
}

export const useAppStore = create<AppState>()(
  devtools(
    subscribeWithSelector(
      persist(
        immer((set) => ({
          theme: "dark",
          sidebarOpen: true,
          recentProjects: [],

          setTheme: (theme) => set((s) => { s.theme = theme; }),
          toggleSidebar: () => set((s) => { s.sidebarOpen = !s.sidebarOpen; }),
          addRecentProject: (id) =>
            set((s) => {
              s.recentProjects = [id, ...s.recentProjects.filter((p) => p !== id)].slice(0, 10);
            }),
        })),
        {
          name: "app-storage",
          partialize: (s) => ({
            theme: s.theme,
            recentProjects: s.recentProjects,
          }),
        },
      ),
    ),
    { name: "AppStore" },
  ),
);
```

### Slice 模式（大 Store 拆分）

```tsx
// stores/createAuthSlice.ts
interface AuthSlice {
  token: string | null;
  isAuthenticated: boolean;
  login: (token: string) => void;
  logout: () => void;
}

const createAuthSlice: StateCreator<AppState, [], [], AuthSlice> = (set) => ({
  token: null,
  isAuthenticated: false,
  login: (token) => set({ token, isAuthenticated: true }),
  logout: () => set({ token: null, isAuthenticated: false }),
});

// stores/createUISlice.ts
interface UISlice {
  sidebarOpen: boolean;
  toggleSidebar: () => void;
}

const createUISlice: StateCreator<AppState, [], [], UISlice> = (set) => ({
  sidebarOpen: true,
  toggleSidebar: () => set((s) => ({ sidebarOpen: !s.sidebarOpen })),
});

// stores/useAppStore.ts — 合并
type AppState = AuthSlice & UISlice;

export const useAppStore = create<AppState>()((...a) => ({
  ...createAuthSlice(...a),
  ...createUISlice(...a),
}));
```

### 订阅模式（组件外监听）

```tsx
// 在组件外监听状态变化（类似 Pub/Sub）
const unsub = useAppStore.subscribe(
  (state) => state.isAuthenticated,
  (isAuth, wasAuth) => {
    if (!isAuth && wasAuth) {
      router.navigate("/login");
    }
  },
  { fireImmediately: false },
);
```

### 异步操作

```tsx
const useProjectStore = create<ProjectState>((set, get) => ({
  projects: [],
  isLoading: false,
  error: null,

  fetchProjects: async () => {
    set({ isLoading: true, error: null });
    try {
      const data = await api.getProjects();
      set({ projects: data, isLoading: false });
    } catch (error) {
      set({ error: (error as Error).message, isLoading: false });
    }
  },

  // 乐观更新
  deleteProject: async (id: string) => {
    const prev = get().projects;
    // 乐观：先删除
    set({ projects: prev.filter((p) => p.id !== id) });
    try {
      await api.deleteProject(id);
    } catch {
      // 回滚
      set({ projects: prev });
    }
  },
}));
```

## 四、Jotai — 原子化状态

### 何时选 Jotai 而非 Zustand

```
Zustand：全局 Store，适合「几块大状态」
Jotai：  原子散装，适合「很多小状态 + 互相派生」

具体场景：
  - 编辑器中多个独立的 UI 控制项（每个是一个 atom）
  - 搜索过滤：query atom → derived filtered results atom
  - 多选/勾选：每个项的选中状态独立
```

### 核心 API

```tsx
import { atom, useAtom, useAtomValue, useSetAtom } from "jotai";
import { atomWithStorage } from "jotai/utils";

// 基础 atom
const countAtom = atom(0);

// 持久化 atom
const themeAtom = atomWithStorage<"light" | "dark">("theme", "dark");

// 派生 atom（只读）
const doubleCountAtom = atom((get) => get(countAtom) * 2);

// 异步派生 atom
const searchResultsAtom = atom(async (get) => {
  const query = get(queryAtom);
  if (!query) return [];
  return await searchAPI(query);
});

// 可写派生 atom（action）
const incrementAtom = atom(null, (get, set) => {
  set(countAtom, get(countAtom) + 1);
});

// 使用
function Counter() {
  const [count, setCount] = useAtom(countAtom);
  const double = useAtomValue(doubleCountAtom);
  const increment = useSetAtom(incrementAtom);
  // ...
}
```

## 五、Valtio — 可变代理

### 核心理念

> 像写普通 JS 对象一样写状态，Valtio 自动追踪变化

```tsx
import { proxy, useSnapshot } from "valtio";

const state = proxy({
  count: 0,
  todos: [] as string[],
});

// 直接修改（像 Vue 一样）
function increment() {
  state.count++;
}

function addTodo(text: string) {
  state.todos.push(text);
}

// 使用
function Counter() {
  const snap = useSnapshot(state);
  return <span>{snap.count}</span>;  // 自动精确 re-render
}
```

### 适合

- 从 Vue 转 React 的开发者
- 快速原型
- 不喜欢 immutable update 的场景

### 不适合

- SSR（proxy 在服务端行为不一致）
- 需要时间旅行调试

## 六、TanStack Query — 服务端状态

> 不是状态管理库，而是**异步状态同步库**

### 解决的问题

```
以前（手写）：
  useState(loading) + useState(error) + useState(data) + useEffect(fetch)
  + 缓存？手动 Map
  + 刷新？手动 setInterval
  + 取消？手动 AbortController
  + 重试？手动 try/catch + setTimeout
  + 分页？手动 page state

现在（TanStack Query）：
  const { data, isLoading, error } = useQuery({ queryKey, queryFn })
  自动：缓存、后台刷新、重试、取消、分页、乐观更新、SSR、DevTools
```

### 基础用法

```tsx
import { useQuery, useMutation, useQueryClient } from "@tanstack/react-query";

// 查询
function ProjectList() {
  const { data, isLoading, error } = useQuery({
    queryKey: ["projects"],
    queryFn: () => api.getProjects(),
    staleTime: 5 * 60 * 1000,  // 5 分钟内视为新鲜
  });
  // ...
}

// 变更 + 自动刷新
function CreateProject() {
  const queryClient = useQueryClient();

  const mutation = useMutation({
    mutationFn: (data: CreateProjectInput) => api.createProject(data),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ["projects"] });
    },
  });
  // ...
}
```

## 七、持久化策略

| 层级 | 存储位置 | 方式 | 示例 |
|------|---------|------|------|
| **UI 偏好** | localStorage | Zustand persist | theme / sidebarOpen |
| **认证 token** | localStorage | Zustand persist (加密) | JWT / refresh token |
| **URL 状态** | URL searchParams | react-router | 筛选条件 / 分页 / Tab |
| **表单草稿** | sessionStorage | React Hook Form + storage | 编辑中的表单 |
| **大量结构化数据** | IndexedDB | Dexie.js | 项目列表 / 聊天历史 |
| **服务端缓存** | 内存 (TanStack) | staleTime + gcTime | API 响应缓存 |

### Zustand 持久化详解

```tsx
persist(
  (set) => ({ /* state */ }),
  {
    name: "app-storage",                      // localStorage key
    storage: createJSONStorage(() => localStorage),  // 可换 sessionStorage
    partialize: (s) => ({                     // 只持久化部分字段
      theme: s.theme,
      recentProjects: s.recentProjects,
    }),
    version: 2,                               // 版本号
    migrate: (persisted, version) => {        // 版本迁移
      if (version === 1) {
        // v1 → v2 迁移逻辑
        return { ...persisted, newField: "default" };
      }
      return persisted;
    },
    onRehydrateStorage: () => (state) => {    // 恢复完成回调
      console.log("State rehydrated", state);
    },
  },
),
```

## 八、本项目状态分层策略

```
┌─────────────────────────────────────────────────────────────────┐
│  Layer 1: Server State — TanStack Query (建议引入)              │
│  → API 响应缓存、后台刷新、分页、乐观更新                      │
│  → 替代手写的 loading/error/data useState                       │
│  → queryKey 做缓存键，staleTime 控制新鲜度                     │
└─────────────────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────────────────┐
│  Layer 2: Global Client State — Zustand (已在用)                │
│  → Auth 状态 (token, isAuthenticated)                           │
│  → UI 偏好 (theme, sidebarOpen, recentProjects)                │
│  → persist middleware → localStorage                            │
│  → subscribeWithSelector → 组件外监听                           │
└─────────────────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────────────────┐
│  Layer 3: Feature State — Zustand (per-feature store)           │
│  → 各 feature 内部的 store (workbench, workspace)              │
│  → 不跨 feature 共享，只在 feature 内使用                      │
└─────────────────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────────────────┐
│  Layer 4: Local State — useState / useReducer                   │
│  → 组件内部的临时状态（isOpen, inputValue 等）                  │
│  → 不持久化，不全局化                                           │
└─────────────────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────────────────┐
│  Layer 5: Form State — React Hook Form (按需)                   │
│  → 复杂表单（多字段验证、动态字段、嵌套）                      │
│  → 简单表单用受控组件即可                                       │
└─────────────────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────────────────┐
│  Layer 6: URL State — react-router searchParams                 │
│  → 筛选条件、分页、当前 Tab                                    │
│  → 用户可以分享/收藏带状态的 URL                                │
└─────────────────────────────────────────────────────────────────┘
```

### Store 文件组织

```
src/
├── features/
│   ├── auth/
│   │   └── hooks/
│   │       └── useAuth.ts          # Auth feature store (Zustand)
│   ├── workbench/
│   │   └── hooks/
│   │       └── useWorkbench.ts     # Workbench feature store
│   └── workspace/
│       └── hooks/
│           └── useWorkspace.ts     # Workspace feature store
└── stores/                          # 仅放跨 feature 的全局 store
    └── useAppStore.ts              # UI 偏好、全局设置
```

## 九、选型决策树

```
你需要管理什么状态？

服务端数据（API 响应）？
  → TanStack Query

全局 UI 状态（theme / sidebar / auth）？
  → Zustand

很多小的、互相派生的原子状态？
  → Jotai（但如果 Zustand 够用就别加）

组件内临时状态？
  → useState / useReducer

复杂流程状态（多步骤/状态机）？
  → XState (已有文档)

表单状态？
  → React Hook Form（复杂表单） / 受控组件（简单表单）

需要 URL 可分享？
  → searchParams + react-router
```
