## 一、2026 年 UI/UX 设计趋势

### 1. Dark Glassmorphism（暗色玻璃态）

> 2026 年定义性视觉趋势。不是简单的 Dark Mode，而是玻璃质感 + 暗色 + 层次感

- **核心要素**：
  - 半透明背景 (`background: rgba(15, 29, 50, 0.6)`)
  - 背景模糊 (`backdrop-blur-xl`)
  - 微妙边框 (`border: 1px solid rgba(255,255,255,0.1)`)
  - 多层叠加（前景/中景/背景各有不同透明度）
- **关键失误**：纯黑背景上玻璃效果不可见，必须有渐变/图案作为底层
- **参考**：[Dark Glassmorphism: The Aesthetic That Will Define UI in 2026](https://medium.com/@developer_89726/dark-glassmorphism-the-aesthetic-that-will-define-ui-in-2026-93aa4153088f)

### 2. Aurora UI（极光渐变）

- 多色渐变融合，模拟北极光
- 色彩在色轮上相邻或形成对比，但必须和谐
- 常用于 Hero Section、背景、CTA 按钮
- 配合 `backdrop-blur` 实现深度感

### 3. AI-era 纹理美学

> 设计不再追求"清晰与控制"，而是拥抱纹理、直觉、亚文化深度

- **告别同质化**：每个 SaaS Dashboard 都长一样的时代结束
- **新方向**：
  - 噪点纹理 (grain/noise overlay)
  - 手绘/有机元素混入数字界面
  - 亚文化视觉语言（brutalism、retro-futurism）
  - 纹理作为信息层级的表达手段
- **参考**：[Aesthetics in the AI Era](https://medium.com/design-bootcamp/aesthetics-in-the-ai-era-visual-web-design-trends-for-2026-5a0f75a10e98)

### 4. 功能性极简 (Functional Minimalism)

- 不是"少即是多"的教条极简
- 而是**每个视觉元素都有功能目的**
- Spotify 式界面：干净布局、直觉操控、零杂乱
- Logo 和品牌趋向极简：定制字体、微妙连字、几何抽象
- **参考**：[12 UI/UX Design Trends That Will Dominate 2026](https://www.index.dev/blog/ui-ux-design-trends)

### 5. 空间 UI (Spatial UI)

- 从 2D 平面向微妙的 3D 层次过渡
- 不是全面 3D，而是通过阴影/透视/层叠暗示空间感
- 卡片、面板、模态框有明确的"高度"概念

### 6. 动态排版 (Kinetic Typography)

- 文字本身成为交互元素
- 滚动驱动的文字变形/显现
- 可变字体 (Variable Fonts) + 动画
- **参考**：[What's Next: 7 UI Design Trends of 2026](https://blog.tubikstudio.com/ui-design-trends-2026/)

### 7. 视觉饱和的反面

> 2026 的专业用户界面（尤其是 EDA/工程工具）正在回归 **"蓝图美学"**

- 深色底 + 技术感线条
- 数据密集但层次清晰
- 信息架构 > 装饰性动画
- **参考**：Tubik Studio 为 Alphane Labs 做的蓝图风格界面

## 三、审美直觉训练方法

> 来源：多个设计博客 & Reddit 设计社区 2026 讨论

### 1. 日常浸泡法

```
每天 15 分钟：
  → Godly/Awwwards 浏览 10 个站点
  → 不分析，只感受：什么让你停下来？
  → 截图收藏感觉好的细节（不是整页）
  → 周末回顾，归纳"为什么好看"
```

### 2. 拆解真实产品

```
选一个你每天用的 App（Spotify / Linear / Notion / Figma）：
  → 截图每个状态：空状态 / 加载中 / 有数据 / 错误
  → 记录：间距比例、字号层级、颜色数量
  → 问：如果去掉一个元素，会不会更好？
```

### 3. 约束练习

```
用 3 种颜色 + 1 种字体 设计一个完整页面
  → 限制迫使你关注层次而非装饰
  → 先灰度设计，最后才加颜色
  → 间距是最容易被忽视的层次工具
```

### 4. "为什么看起来廉价"检查清单

| 问题 | 原因 | 修复 |
|------|------|------|
| 对齐不一致 | 多个对齐基线 | 用 4px/8px 网格严格对齐 |
| 颜色太多 | >5 种主色 | 缩减到 1 主色 + 1 强调色 + 灰色系 |
| 字号层级混乱 | 随意选字号 | 用 Type Scale（1.25/1.333 比率） |
| 间距不均匀 | 随意 padding | 统一 spacing scale (4/8/12/16/24/32/48) |
| 边框太多 | 用线分隔一切 | 用间距和背景色分隔，减少线条 |
| 阴影生硬 | 单一 `box-shadow` | 多层柔和阴影，降低不透明度 |
| 元素太拥挤 | 缺乏留白 | 增加 padding，内容不等于填满 |

## 五、本项目 (CGRA EDA) 视觉策略

### 定位
- 工程工具，用户是专业开发者/芯片设计师
- 暗色主题为主（长时间使用护眼）
- 数据密集，信息架构 > 装饰性动画
- 科技感 + 功能性极简

### 视觉体系

```
┌─ 基调 ─────────────────────────────────────┐
│  Dark Glassmorphism + 蓝图美学              │
│  深蓝黑底色 (#09111f → #0f1728)            │
│  毛玻璃面板 (bg-surface + backdrop-blur)    │
│  微妙发光边框 (border-white/10)             │
└────────────────────────────────────────────┘
┌─ 色彩 ─────────────────────────────────────┐
│  主色: #4f7cff (蓝紫) — 操作/焦点          │
│  强调: #2bc5ff / #00ffff (青) — 高亮/链接   │
│  文字: #e8eef7 (主) / #9db0ca (次)         │
│  成功: 绿 / 警告: 琥珀 / 错误: 红          │
│  表面: rgba(12, 19, 34, 0.76) + blur       │
└────────────────────────────────────────────┘
┌─ 动画 ─────────────────────────────────────┐
│  原则: 服务于 UX，不做装饰性动画           │
│  加载: Shimmer / Skeleton (animate-pulse)  │
│  过渡: 200ms ease-out (opacity + transform)│
│  AI 响应: FlowToken 逐词淡入              │
│  尊重: prefers-reduced-motion              │
└────────────────────────────────────────────┘
┌─ 排版 ─────────────────────────────────────┐
│  字体: system-ui (Segoe UI / PingFang SC)  │
│  Scale: 1.25 Major Third                   │
│  正文: 14px / 标题: 18/24/32px             │
│  行高: 1.5 (正文) / 1.2 (标题)            │
└────────────────────────────────────────────┘
```
