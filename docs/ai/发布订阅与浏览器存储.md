### 一、Event Emitter 库对比

| 库 | 体积 | TypeScript | 通配符 | once | 性能 | 特色 | 周下载量 |
|----|------|-----------|--------|------|------|------|----------|
| **mitt** | ~200B | ✅ 原生 | ✅ `*` | ❌ (手动) | ⭐⭐⭐⭐ | 极致轻量，Preact 作者出品 | ~600万 |
| **nanoevents** | ~100B | ✅ 原生 | ❌ | ❌ | ⭐⭐⭐⭐⭐ | 最小体积，现代 ESM | ~80万 |
| **eventemitter3** | ~1KB | ✅ (社区) | ❌ | ✅ | ⭐⭐⭐⭐ | 最成熟，Node EventEmitter API 兼容 | ~2500万 |
| **tseep** | ~2KB | ✅ 原生 | ❌ | ✅ | ⭐⭐⭐⭐⭐ | 最快（x12 于 eventemitter3），零分配 | ~100万 |
| **tiny-emitter** | ~500B | ❌ (社区) | ❌ | ✅ | ⭐⭐⭐ | 简单均衡 | ~800万 |
| **emittery** | ~3KB | ✅ 原生 | ❌ | ✅ | ⭐⭐⭐ | 异步 emit、unsubscribe token | ~500万 |
| **pubsub-js** | ~1KB | ✅ (社区) | ✅ 层级 | ❌ | ⭐⭐⭐ | 经典 Pub/Sub 模式，topic 层级 | ~30万 |
| **RxJS Subject** | 重 | ✅ 原生 | — | — | ⭐⭐⭐ | 完整响应式流，操作符丰富 | ~3000万 |

### 二、推荐选择

```
极致体积（<200B）→ mitt 或 nanoevents
极致性能（高频事件）→ tseep
Node.js API 兼容 → eventemitter3
异步事件 → emittery
需要 filter/map/debounce → RxJS Subject
```

### 三、mitt — 本项目推荐

> 200 字节，TypeScript 原生，Preact 作者（developit）出品

```bash
npm install mitt
```

```tsx
import mitt from "mitt";

// 1. 定义事件类型
type Events = {
  "task:start": { taskId: string };
  "task:complete": { taskId: string; result: unknown };
  "task:error": { taskId: string; error: string };
  "notification": { type: "success" | "error"; message: string };
};

// 2. 创建 emitter
const emitter = mitt<Events>();

// 3. 监听
emitter.on("task:complete", ({ taskId, result }) => {
  console.log(`任务 ${taskId} 完成`, result);
});

// 监听所有事件（通配符）
emitter.on("*", (type, event) => {
  console.log(`[EventBus] ${type}`, event);
});

// 4. 发送
emitter.emit("task:start", { taskId: "compile-001" });

// 5. 取消监听
const handler = (e: Events["notification"]) => toast[e.type](e.message);
emitter.on("notification", handler);
emitter.off("notification", handler);
```

### 四、tseep — 极致性能

> 号称世界最快 EventEmitter（2024 benchmark），TypeScript 原生

```bash
npm install tseep
```

```tsx
import { EventEmitter } from "tseep";

interface MyEvents {
  data: (payload: { id: number; value: string }) => void;
  error: (error: Error) => void;
  close: () => void;
}

const ee = new EventEmitter<MyEvents>();

ee.on("data", (payload) => {
  console.log(payload.id, payload.value); // 完全类型安全
});

ee.once("close", () => {
  console.log("连接关闭");
});

ee.emit("data", { id: 1, value: "hello" });
```

### 五、nanoevents — 最小体积

> ~100 字节，ESM only，适合现代项目

```bash
npm install nanoevents
```

```tsx
import { createNanoEvents } from "nanoevents";

interface Events {
  tick: (count: number) => void;
  error: (message: string) => void;
}

const emitter = createNanoEvents<Events>();

const unbind = emitter.on("tick", (count) => {
  console.log(`Tick: ${count}`);
});

emitter.emit("tick", 42);

// 取消监听（返回的函数直接调用）
unbind();
```

### 六、React 集成模式

#### 模式 1：全局 EventBus 单例

```tsx
// utils/eventBus.ts
import mitt from "mitt";
import type { Events } from "@/types/events";

export const eventBus = mitt<Events>();

// hooks/useEventBus.ts
import { useEffect } from "react";
import { eventBus } from "@/utils/eventBus";
import type { Events } from "@/types/events";

export function useEvent<K extends keyof Events>(
  event: K,
  handler: (payload: Events[K]) => void,
) {
  useEffect(() => {
    eventBus.on(event, handler);
    return () => eventBus.off(event, handler);
  }, [event, handler]);
}

// 组件中使用
function TaskMonitor() {
  useEvent("task:complete", ({ taskId }) => {
    toast.success(`任务 ${taskId} 完成`);
  });
  return <div>监控中...</div>;
}
```

#### 模式 2：Zustand subscribe（本项目已用 Zustand）

```tsx
// 利用 Zustand 的 subscribe 做 Pub/Sub
import { useEffect } from "react";
import { useAuthStore } from "@/features/auth/hooks/useAuth";

// 监听特定字段变化
useEffect(() => {
  const unsub = useAuthStore.subscribe(
    (state) => state.isAuthenticated,
    (isAuth, wasAuth) => {
      if (!isAuth && wasAuth) {
        toast.error("会话已过期");
      }
    },
  );
  return unsub;
}, []);
```

#### 模式 3：React Context + EventEmitter

```tsx
const EventBusContext = createContext<ReturnType<typeof mitt<Events>> | null>(null);

function EventBusProvider({ children }: PropsWithChildren) {
  const bus = useMemo(() => mitt<Events>(), []);
  return <EventBusContext.Provider value={bus}>{children}</EventBusContext.Provider>;
}

function useEventBus() {
  const bus = useContext(EventBusContext);
  if (!bus) throw new Error("EventBusProvider missing");
  return bus;
}
```

### 七、选型决策

```
本项目现状：Zustand (全局状态) + Sonner (Toast)

推荐方案：
  简单事件通信 → Zustand subscribe (已有，零新增依赖)
  跨模块解耦通信 → mitt (200B，TypeScript 原生)
  高频事件（拖拽/鼠标/流式 token）→ tseep (性能最优)
  复杂流操作（debounce/merge/switchMap）→ RxJS Subject

不推荐：
  eventemitter3 → 适合 Node，浏览器不需要那么多功能
  pubsub-js → 没有 TypeScript 原生支持
  自写 EventBus → mitt/nanoevents 已经足够小，不必重复造轮子
```

### 八、浏览器原生存储 API 对比

| API | 容量上限 | 异步 | 结构化数据 | 索引/查询 | 多标签同步 | WebWorker |
|-----|---------|------|-----------|-----------|-----------|-----------|
| **Cookies** | ~4KB | ❌ | ❌ (string) | ❌ | ✅ (每次请求带上) | ❌ |
| **localStorage** | 5-10MB | ❌ 同步阻塞 | ❌ (string) | ❌ | ✅ storage event | ❌ |
| **sessionStorage** | 5MB | ❌ 同步阻塞 | ❌ (string) | ❌ | ❌ (标签级) | ❌ |
| **IndexedDB** | 磁盘 80% | ✅ | ✅ JSON 原生 | ✅ 索引 | ❌ (需 BroadcastChannel) | ✅ |
| **OPFS** | 磁盘 80% | ✅ | ❌ (二进制) | ❌ | ❌ | ✅ (最佳) |
| **Cache API** | 磁盘 80% | ✅ | ❌ (Request/Response) | ❌ | ❌ | ✅ |
| **WASM SQLite** | 取决于底层 | ✅ | ✅ JSON (3.38+) | ✅ SQL | ❌ | ✅ |

### 九、性能排序（Chrome 基准）

| 操作 | 最快 → 最慢 |
|------|-------------|
| **小写入** | localStorage (0.017ms) > IndexedDB (0.17ms) > OPFS Worker (1.5ms) > SQLite WASM (3ms+) |
| **小读取** | localStorage (0.005ms) > IndexedDB > OPFS > SQLite WASM |
| **大批量写** | OPFS Worker > IndexedDB > SQLite WASM > localStorage (阻塞!) |
| **大批量读** | SQLite WASM (缓存命中) ≈ IndexedDB > OPFS > localStorage |
| **初始化** | localStorage (0ms) > IndexedDB (~10ms) > OPFS (~50ms) > SQLite WASM (~500ms) |

> 数据来源：[RxDB Storage Comparison](https://rxdb.info/articles/localstorage-indexeddb-cookies-opfs-sqlite-wasm.html)

### 十一、Dexie.js — 推荐方案

> "IndexedDB made simple"，API 像写数据库而非回调地狱

```bash
npm install dexie dexie-react-hooks
```

#### 定义数据库

```tsx
// db.ts
import Dexie, { type EntityTable } from "dexie";

interface Project {
  id: string;
  name: string;
  createdAt: Date;
  status: "draft" | "compiling" | "completed";
}

interface ChatMessage {
  id: string;
  projectId: string;
  role: "user" | "assistant";
  content: string;
  timestamp: Date;
}

const db = new Dexie("cgra-eda") as Dexie & {
  projects: EntityTable<Project, "id">;
  messages: EntityTable<ChatMessage, "id">;
};

db.version(1).stores({
  projects: "id, status, createdAt",         // id=主键, status+createdAt=索引
  messages: "id, projectId, timestamp",       // 支持按 projectId 查询
});

export { db };
export type { Project, ChatMessage };
```

#### React 响应式查询

```tsx
import { useLiveQuery } from "dexie-react-hooks";
import { db } from "@/db";

function ProjectList() {
  // 自动响应数据变化，任何地方写入都会触发 re-render
  const projects = useLiveQuery(
    () => db.projects.where("status").equals("completed").toArray(),
  );

  if (!projects) return <Spinner />;
  return (
    <ul>
      {projects.map((p) => (
        <li key={p.id}>{p.name}</li>
      ))}
    </ul>
  );
}
```

#### CRUD 操作

```tsx
// 增
await db.projects.add({
  id: crypto.randomUUID(),
  name: "My Project",
  createdAt: new Date(),
  status: "draft",
});

// 查
const project = await db.projects.get("some-id");
const drafts = await db.projects.where("status").equals("draft").toArray();

// 改
await db.projects.update("some-id", { status: "compiling" });

// 删
await db.projects.delete("some-id");

// 批量
await db.projects.bulkAdd([...]);
await db.projects.bulkPut([...]);

// 事务
await db.transaction("rw", db.projects, db.messages, async () => {
  await db.projects.add(newProject);
  await db.messages.add(initialMessage);
});
```

#### 版本迁移

```tsx
db.version(1).stores({
  projects: "id, status",
});

db.version(2).stores({
  projects: "id, status, createdAt", // 新增索引
}).upgrade((tx) => {
  return tx.table("projects").toCollection().modify((project) => {
    project.createdAt = project.createdAt || new Date();
  });
});
```

### 十二、idb-keyval — 极简 KV

> 600B，最简单的 IndexedDB 封装

```bash
npm install idb-keyval
```

```tsx
import { get, set, del, keys, clear } from "idb-keyval";

// 存
await set("user-prefs", { theme: "dark", lang: "zh" });

// 取
const prefs = await get("user-prefs");

// 删
await del("user-prefs");

// 列出所有 key
const allKeys = await keys();
```

**适合**：用户偏好、缓存 token、简单 KV 存取。不需要查询/索引。

### 十三、RxDB — 重型响应式数据库

> 适合需要**跨端同步 + 响应式 + 复杂查询**的大型应用

- **后端选择**：IndexedDB / OPFS / SQLite WASM / Memory / Worker
- **响应式**：`RxQuery.$.subscribe()` 自动更新
- **同步**：CouchDB / GraphQL / WebSocket / 自定义
- **加密**：内置字段级加密
- **压缩**：Key Compression 减少存储体积

```tsx
import { createRxDatabase } from "rxdb";
import { getRxStorageDexie } from "rxdb/plugins/storage-dexie";

const db = await createRxDatabase({
  name: "cgra-eda",
  storage: getRxStorageDexie(),
});

await db.addCollections({
  projects: {
    schema: {
      version: 0,
      primaryKey: "id",
      type: "object",
      properties: {
        id: { type: "string", maxLength: 36 },
        name: { type: "string" },
        status: { type: "string" },
      },
      required: ["id", "name"],
    },
  },
});

// 响应式查询
db.projects.find({ selector: { status: "draft" } }).$.subscribe((docs) => {
  console.log("Drafts updated:", docs);
});
```

**注意**：RxDB 体积较大（~45KB+），有商业许可要求。只在需要复杂同步/离线能力时使用。

### 十四、选型决策树

```
只需要存几个 key-value（偏好/token/小状态）？
  ├─ 同步访问够用 → localStorage (Zustand persist 已在用)
  └─ 异步/稍大 → idb-keyval (600B)

需要结构化数据 + 索引查询（项目/消息/历史记录）？
  ├─ 中等复杂 → Dexie.js ⭐（本项目首推）
  └─ 重型（同步/加密/多后端）→ RxDB

需要 SQL 查询能力？
  → SQLite WASM (sql.js / wa-sqlite)
  → 注意：初始化慢 ~500ms，WASM 文件 ~1MB

需要存大文件（图片/音频/二进制）？
  → OPFS 或 Cache API

多标签页同步？
  → BroadcastChannel API + 任意存储
  → 或 Dexie.js useLiveQuery（自动跨标签响应）
```
