## 二、实时推送 & 通知基础设施

### 客户端方案

| 方案 | 网址 | 类型 | 适用 |
|------|------|------|------|
| **Socket.IO** | https://socket.io | WebSocket 封装 | 双向实时通信、聊天、协作 |
| **Web Push API** | MDN Web Push | 浏览器原生推送 | 离线推送、后台通知 |
| **SSE (Server-Sent Events)** | — | 单向服务端推送 | AI 流式响应、实时日志 |
| **Knock** | https://knock.app | 通知基础设施 | 全渠道通知（in-app + email + push + SMS） |

### SSE vs WebSocket 选型

```
SSE (EventSource):
  ✅ 单向推送（服务端 → 客户端）
  ✅ 自动重连
  ✅ HTTP/2 友好
  ✅ 简单轻量
  ❌ 不支持双向通信
  → 适合：AI 流式响应、实时日志、通知推送

WebSocket:
  ✅ 全双工双向通信
  ✅ 低延迟
  ❌ 需要额外的连接管理
  ❌ 代理/防火墙可能有问题
  → 适合：聊天、协作编辑、游戏
```

## 三、消息展示与恢复模式

### 3.1 消息持久化 (Zustand Persist)

```tsx
import { create } from "zustand";
import { persist } from "zustand/middleware";

interface MessageStore {
  messages: Message[];
  addMessage: (msg: Message) => void;
  clearMessages: () => void;
}

const useMessageStore = create<MessageStore>()(
  persist(
    (set) => ({
      messages: [],
      addMessage: (msg) =>
        set((s) => ({ messages: [...s.messages, msg] })),
      clearMessages: () => set({ messages: [] }),
    }),
    {
      name: "chat-messages", // localStorage key
      // 可选：自定义序列化、版本迁移
    },
  ),
);

// 页面刷新后自动从 localStorage 恢复 messages
```

### 3.2 Optimistic UI (乐观更新)

> 先更新 UI，再异步提交服务端，失败则回滚

```tsx
import { useOptimistic } from "react";

function MessageList({ messages, sendMessage }) {
  const [optimisticMessages, addOptimistic] = useOptimistic(
    messages,
    (state, newMsg) => [...state, { ...newMsg, pending: true }],
  );

  async function handleSend(content: string) {
    const tempMsg = { id: crypto.randomUUID(), content, pending: true };
    addOptimistic(tempMsg);
    
    try {
      await sendMessage(content); // 服务端请求
    } catch {
      toast.error("发送失败，请重试");
      // React 自动回滚 optimistic state
    }
  }

  return (
    <div>
      {optimisticMessages.map((msg) => (
        <div key={msg.id} className={msg.pending ? "opacity-60" : ""}>
          {msg.content}
        </div>
      ))}
    </div>
  );
}
```

### 3.3 事件驱动 React 架构

> 解耦组件间通信，避免 props drilling

**方案 1：自定义 EventEmitter**

```tsx
// utils/eventBus.ts
type Listener = (...args: unknown[]) => void;

class EventBus {
  private listeners = new Map<string, Set<Listener>>();

  on(event: string, fn: Listener) {
    if (!this.listeners.has(event)) this.listeners.set(event, new Set());
    this.listeners.get(event)!.add(fn);
    return () => this.listeners.get(event)?.delete(fn);
  }

  emit(event: string, ...args: unknown[]) {
    this.listeners.get(event)?.forEach((fn) => fn(...args));
  }
}

export const bus = new EventBus();

// 组件 A：发送
bus.emit("notification:new", { type: "success", message: "已保存" });

// 组件 B：监听
useEffect(() => {
  const off = bus.on("notification:new", (data) => {
    toast.success(data.message);
  });
  return off;
}, []);
```

**方案 2：event-driven-react (npm)**

- GitHub: https://github.com/yozibak/event-driven-react
- 提供 `useEvent` / `useEmit` hooks，类型安全

**方案 3：Zustand (推荐)**

- 本项目已使用 Zustand，可直接用 `subscribe` 实现事件订阅
- 结合 `persist` middleware 实现消息恢复

## 四、AI 任务队列 & 进度展示

> shadcn/ui AI 提供的专用组件，适合展示 AI Agent 多步任务进度

| 组件 | 网址 | 功能 |
|------|------|------|
| **Queue** | https://www.shadcn.io/ai/queue | 多任务队列 + 进度可视化 |
| **Plan** | https://www.shadcn.io/ai/plan | 可展开计划展示 + 步骤追踪 |
| **Task** | https://www.shadcn.io/ai/task | 任务列表 + 文件引用 + 完成状态 |
| **Loader** | https://www.shadcn.io/ai/loader | 多尺寸动画加载指示器 |
| **Shimmer** | https://www.shadcn.io/ai/shimmer | 流式内容 skeleton 脉冲 |

### 使用场景

```
用户发送 prompt
  → 显示 Loader（等待 AI 响应）
  → AI 开始执行 → 显示 Plan（分步计划）
  → 每步执行中 → 显示 Task（当前步骤 + 进度）
  → 多步并行 → 显示 Queue（队列进度）
  → 步骤完成 → 流式输出结果 → Shimmer → 文字渐现
```

### 本项目推荐方案

```
┌─────────────────────────────────────────────────┐
│  Toast 通知层                                    │
│  Sonner (已集成)                                 │
│  - 操作反馈 (success / error / promise)          │

         ↑ bus.emit("toast", ...)
┌─────────────────────────────────────────────────┐
│  事件总线                                        │
│  Zustand subscribe / EventBus                    │
│  - 跨组件通信                                    │
│  - 解耦发送方与展示方                             │
└─────────────────────────────────────────────────┘
         ↑ WebSocket / SSE
┌─────────────────────────────────────────────────┐
│  实时通信层                                      │
│  SSE (AI 流式) / WebSocket (协作)                │
│  - AI 响应流 → FlowToken 渲染                    │
│  - 后端推送 → 通知/状态更新                       │
└─────────────────────────────────────────────────┘
         ↑ persist
┌─────────────────────────────────────────────────┐
│  持久化层                                        │
│  Zustand persist → localStorage                  │
│  - 对话历史恢复                                   │
```
