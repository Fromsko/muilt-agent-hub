## 一、扫光 / Shimmer / Skeleton 加载

> 内容加载时的视觉占位反馈，提升感知性能

### 方案对比

| 方案 | 来源 | 效果 | 适用场景 |
|------|------|------|----------|
| **Tailwind `animate-pulse`** | 内置 | 简单脉冲透明度变化 | 快速原型、简单占位 |
| **Tailwind shimmer (自定义)** | `@keyframes` + 线性渐变 | 从左到右扫光 | 卡片/列表骨架屏 |
| **Flowbite Skeleton** | https://flowbite.com/docs/components/skeleton/ | 多种预设骨架模式 | Flowbite 项目 |
| **react-loading-skeleton** | npm: `react-loading-skeleton` | 自动适配容器尺寸的动画骨架 | React 通用 |
| **shadcn AI Shimmer** | https://www.shadcn.io/ai/shimmer | 流式内容脉冲占位 | AI 聊天场景 |
| **Syncfusion Skeleton** | https://ej2.syncfusion.com/react | Wave / Pulse / Fade 三种 shimmer | 企业级 |

### TailwindCSS 自定义 Shimmer 实现

```css
/* globals.css — @layer utilities */
@keyframes shimmer {
  0% { background-position: -200% 0; }
  100% { background-position: 200% 0; }
}

.animate-shimmer {
  background: linear-gradient(
    90deg,
    transparent 0%,
    rgba(255, 255, 255, 0.08) 50%,
    transparent 100%
  );
  background-size: 200% 100%;
  animation: shimmer 1.5s ease-in-out infinite;
}
```

## 三、Aurora / 渐变动画背景

> 模拟北极光的多色渐变融合效果，适合 Hero Section / Landing Page

### 组件库方案

| 方案 | 来源 | 特点 |
|------|------|------|
| **Aceternity Aurora Background** | https://ui.aceternity.com/components/aurora-background | Tailwind + Framer Motion，最流行方案 |
| **shadcn.io Aurora** | https://www.shadcn.io/background/aurora | 纯 CSS 动画，60fps，无重型动画库 |
| **Petal Aurora** | https://petal.build/components/aurora | 可定制颜色/反转的 Aurora 组件 |

### 纯 CSS 实现要点

```css
@keyframes aurora {
  0%, 100% { background-position: 50% 50%; }
  25% { background-position: 0% 100%; }
  50% { background-position: 100% 0%; }
  75% { background-position: 0% 0%; }
}

.aurora-bg {
  background: 
    radial-gradient(ellipse at 20% 50%, rgba(120, 80, 255, 0.3), transparent 50%),
    radial-gradient(ellipse at 80% 20%, rgba(0, 200, 255, 0.2), transparent 50%),
    radial-gradient(ellipse at 50% 80%, rgba(255, 80, 200, 0.15), transparent 50%);
  background-size: 200% 200%;
  animation: aurora 15s ease-in-out infinite;
}
```

### 关键点
- 背景需要有内容可以「透过」——纯黑背景上看不到玻璃效果
- 配合 `backdrop-blur` 做 Dark Glassmorphism 效果
- 使用 `will-change: background-position` 优化 GPU 加速

## 四、边框扫光 / Border Sweep

> 光线沿边框移动的扫描效果，科技感强

### 实现方式

**方式 1：Conic Gradient 旋转**
```css
@keyframes border-sweep {
  from { --angle: 0deg; }
  to { --angle: 360deg; }
}

@property --angle {
  syntax: "<angle>";
  initial-value: 0deg;
  inherits: false;
}

.border-sweep {
  border: 2px solid transparent;
  background-image: 
    linear-gradient(var(--color-background), var(--color-background)),
    conic-gradient(from var(--angle), transparent 60%, #4f7cff 80%, transparent 100%);
  background-origin: border-box;
  background-clip: padding-box, border-box;
  animation: border-sweep 3s linear infinite;
}
```

**方式 2：伪元素旋转渐变**
```css
.sweep-card {
  position: relative;
  overflow: hidden;
  border-radius: 12px;
}
.sweep-card::before {
  content: "";
  position: absolute;
  inset: -2px;
  background: conic-gradient(from 0deg, transparent, #4f7cff, transparent 30%);
  border-radius: inherit;
  animation: spin 3s linear infinite;
  z-index: -1;
}
```

## 六、Spotlight / 聚光灯效果

> 鼠标位置跟随的光照效果

### Aceternity Spotlight

- **组件**：https://ui.aceternity.com/components/spotlight
- **原理**：`onMouseMove` 获取坐标 → 更新 `radial-gradient` 的 `at` 位置
- **性能**：使用 `requestAnimationFrame` + CSS 变量避免 React re-render

### 手动实现核心

```tsx
function Spotlight({ children }: PropsWithChildren) {
  const ref = useRef<HTMLDivElement>(null);

  const handleMouseMove = useCallback((e: MouseEvent) => {
    if (!ref.current) return;
    const rect = ref.current.getBoundingClientRect();
    ref.current.style.setProperty("--x", `${e.clientX - rect.left}px`);
    ref.current.style.setProperty("--y", `${e.clientY - rect.top}px`);
  }, []);

  return (
    <div
      ref={ref}
      onMouseMove={handleMouseMove}
      style={{
        background: `radial-gradient(
          300px circle at var(--x) var(--y),
          rgba(79, 124, 255, 0.15),
          transparent 60%
        )`,
      }}
    >
      {children}
    </div>
  );
}
```

## 七、性能注意事项

1. **只动画 `transform` 和 `opacity`** — GPU 加速属性，不触发 layout/paint
2. **`will-change` 慎用** — 提前告知浏览器优化，但不要滥用（会占用内存）
3. **`prefers-reduced-motion`** — 始终为无障碍提供降级
4. **避免同时动画 > 20 个元素** — 分批或使用虚拟化
5. **CSS 动画优于 JS 动画** — 除非需要 Framer Motion 的手势/spring 物理
6. **`@property` 注册** — CSS Houdini 允许动画化自定义属性（如角度），但需检查浏览器兼容
